# Blog & Personal Website Stock Detection

## ✅ Yes, it works with ANY website including personal blogs!

The system is designed to work with **any publicly accessible website**, including:
- Personal investment blogs
- Medium articles
- WordPress blogs
- Substack newsletters
- Twitter/X threads (via published URLs)
- LinkedIn articles
- Any HTML content

## 🔍 How It Works

### 1. **Universal Content Extraction**
The system searches these HTML elements (works for most blog platforms):
```
- <article> tags (standard blog posts)
- .article, .post, .blog-post classes (common blog structures)
- <h1>, <h2>, <h3> headings
- .headline, .title classes
- Links containing "article", "post", "blog"
- Any text content in the page body
```

### 2. **Intelligent Stock Matching**
- Uses **word boundary detection** (`\b`) to match exact stock symbols
- Example: "TCS" matches but "TCSL" doesn't
- Case-insensitive matching
- Searches full page content, not just headlines

### 3. **Context Extraction**
For each stock mention found:
- Extracts the headline/title
- Captures a **snippet** (50 chars before + 100 chars after the stock mention)
- Finds the article URL if available
- Shows which stocks were mentioned

## 🎯 Best Practices

### For Better Results:

1. **Add the main page URL**
   - Good: `https://investmentblog.com/posts`
   - Also Good: `https://investmentblog.com/2024/stock-analysis`

2. **Stock symbols should match exactly**
   - Your portfolio: "RELIANCE"
   - Blog mentions: "RELIANCE" or "Reliance" or "reliance" ✅
   - Blog mentions: "RIL" ❌ (different symbol)

3. **Public content only**
   - Works: Any publicly accessible URL
   - Doesn't work: Password-protected blogs, paywalls

### Limitations:

❌ **Won't work if:**
- Website blocks automated access (robots.txt)
- Content is behind a login/paywall
- Website uses heavy JavaScript rendering (some dynamic sites)
- Website has anti-scraping protection

⚠️ **May have issues with:**
- Very dynamic sites (heavy React/Vue apps)
- Sites with aggressive rate limiting
- Sites requiring authentication

```javascript
// Word boundary regex ensures exact matches
const regex = new RegExp(`\\b${stock}\\b`, 'i');

// Example:
Stock: "TCS"
```

## 🎓 Tips for Blog Owners

If you run an investment blog and want to be easily searchable:

1. **Use standard HTML structure**
   ```html
   <article>
     <h1>Title</h1>
     <div class="content">Your analysis...</div>
   </article>
   ```

2. **Use stock symbols consistently**
   - Good: "RELIANCE", "TCS", "INFY"
   - Also mention full names: "Reliance Industries (RELIANCE)"

3. **Keep content public**
   - Avoid paywalls for better discoverability

4. **Use semantic HTML**
   - Proper heading tags
   - Article/section tags
   - Clear content structure

## ❓ FAQ

**Q: Will it work with my personal WordPress blog?**
A: Yes! As long as it's publicly accessible.

**Q: What about paywalled content?**
A: No, the system can only read publicly visible content.

**Q: Can it search PDF files?**
A: Not currently, only HTML content.

**Q: Does it work with Twitter/X threads?**
A: If the thread has a public URL, yes!

**Q: Will frequent searching get me blocked?**
A: We use proper headers and delays to be respectful. Don't search the same site more than once per hour.

**Q: Can I add multiple blogs?**
A: Yes! Add as many sources as you want.

**Q: What if my stock symbol is also a common word?**
A: Use word boundaries to avoid false matches. If issues persist, we can add symbol validation.
